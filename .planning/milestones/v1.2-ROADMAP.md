## Phase Details

### Phase 13: Process Foundation
**Goal**: App can execute a GSD command as a child process with live-streamed output and reliable cancellation — headless, no UI yet
**Depends on**: Phase 12
**Requirements**: PROC-01, PROC-02, PROC-03, PROC-04, PROC-05, SAFE-01
**Success Criteria** (what must be TRUE):
  1. User can initiate a GSD command from the app and it runs in the correct project directory
  2. App resolves the claude CLI path on a fresh macOS account where claude is not on the launchd PATH
  3. Live output lines appear in memory as the command runs — no waiting for process exit
  4. User can cancel a running command and no orphaned processes remain in Activity Monitor
  5. Only one command runs per project at a time — a second trigger on the same project is blocked while the first runs
**Plans:** 3/3 plans complete
Plans:
- [ ] 13-01-PLAN.md — Models (CommandState, CommandRun) + ShellEnvironmentService + GSDOutputParser
- [ ] 13-02-PLAN.md — ProcessActor with PTY, live streaming, and cancellation
- [ ] 13-03-PLAN.md — CommandRunnerService with per-project queues + CommandHistoryStore

### Phase 14: Output Panel
**Goal**: Users can see live command output, distinguish errors, know when a command succeeded or failed, and toggle to structured GSD output — all in an output panel in the app
**Depends on**: Phase 13
**Requirements**: OUTP-01, OUTP-02, OUTP-03, OUTP-04, OUTP-05, OUTP-06, SAFE-03, SAFE-04
**Success Criteria** (what must be TRUE):
  1. User sees live-streamed output lines auto-scrolling as the command runs in a panel within the app
  2. User sees stderr lines in a distinct color (Gruvbox red) versus stdout lines
  3. User sees the exit code and a clear success/failure signal when the command finishes
  4. User can toggle between raw output and a structured view that parses GSD banners, task status, and progress markers
  5. User sees a confirmation dialog before cancelling a running command, and an actionable error message with recovery suggestions on failure
**Plans:** 2/2 plans complete
Plans:
- [ ] 14-01-PLAN.md — Layout foundation: HSplitView, OutputPanelView, OutputRawView with auto-scroll, stderr coloring, rolling buffer
- [ ] 14-02-PLAN.md — Structured view, completion banners, segmented toggle, cancel dialog, failure notifications

### Phase 15: Command Triggering & Integration
**Goal**: Users can trigger GSD commands from context buttons on phases and plans, from the Cmd+K palette, and review command history — with FSEvents reloads suppressed during active runs
**Depends on**: Phase 14
**Requirements**: TRIG-01, TRIG-02, TRIG-03, TRIG-04, SAFE-02
**Success Criteria** (what must be TRUE):
  1. User sees context buttons on phase cards and plan cards and can trigger a GSD command from them
  2. User sees a running state indicator (with elapsed timer) on any surface where a command is active for that project
  3. User can open Cmd+K and trigger GSD commands from the command palette
  4. User can view previous command runs with timestamps and exit codes and re-run any of them
  5. Project state does not reload from disk repeatedly while a command is actively writing to .planning/ files
**Plans:** 3/3 plans complete
Plans:
- [ ] 15-01-PLAN.md — Context buttons on phase cards/plan rows + running state indicators
- [ ] 15-02-PLAN.md — Command palette redesign (multi-step GSD command launcher)
- [ ] 15-03-PLAN.md — Command history view + FSEvents suppression
