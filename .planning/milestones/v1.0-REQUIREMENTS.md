# Requirements: GSD Monitor

**Defined:** 2026-02-13
**Core Value:** Et live, grafisk overblik over alle GSD-projekter på tværs af ~/Developer — så du altid ved hvor du er, uden at åbne en terminal.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Navigation

- [x] **NAV-01**: User can see all GSD-projekter i en sidebar med auto-scan af ~/Developer for .planning/ mapper
- [x] **NAV-02**: User can tilføje projektmapper manuelt (uden for ~/Developer)
- [ ] **NAV-03**: User can søge og filtrere projekter i sidebar
- [ ] **NAV-04**: User can navigere med keyboard shortcuts (Cmd+K command palette)
- [x] **NAV-05**: App auto-discoverer nye projekter når .planning/ mapper oprettes

### Roadmap

- [ ] **ROAD-01**: User can se visuel roadmap med fase-kort (cards) der viser mål, requirements og progress-bar
- [ ] **ROAD-02**: User can se completion-procent per fase og samlet projekt-progress
- [ ] **ROAD-03**: User can drill into en fase for at se detaljeret plan med tasks og status
- [ ] **ROAD-04**: User can se basal markdown-rendering i fase-kort (bold, links, kode)

### Requirements Tracking

- [ ] **REQ-01**: User can se requirements-visning med REQ-ID'er og completion-status
- [ ] **REQ-02**: User can klikke på et REQ-ID for at se definition, hvilken fase det tilhører og status
- [ ] **REQ-03**: User can se cross-reference mellem requirements og plans

### File Monitoring

- [x] **MON-01**: App opdaterer automatisk via FSEvents når .planning/ filer ændres på disk
- [x] **MON-02**: App håndterer event coalescing korrekt (batch-opdateringer)
- [x] **MON-03**: App ekskluderer .git/ og andre irrelevante mapper fra monitoring

### Notifications

- [ ] **NOTF-01**: User modtager macOS system-notifikationer når faser skifter status
- [ ] **NOTF-02**: User modtager notifikation når tasks færdiggøres
- [ ] **NOTF-03**: Notifikationer respekterer macOS Focus modes

### Editor Integration

- [ ] **EDIT-01**: User can klikke på en fil for at åbne den i Cursor eller VS Code
- [ ] **EDIT-02**: App auto-detecter installerede editors (Cursor, VS Code, Zed)
- [ ] **EDIT-03**: User can vælge foretrukken editor i indstillinger

### Parsing

- [x] **PARSE-01**: App parser ROADMAP.md korrekt (faser, mål, requirements, success criteria)
- [x] **PARSE-02**: App parser STATE.md korrekt (projektstatus, aktuel fase)
- [x] **PARSE-03**: App parser REQUIREMENTS.md korrekt (REQ-ID'er, kategorier, status, traceability)
- [x] **PARSE-04**: App parser PLAN.md filer korrekt (tasks, status, wave-struktur)
- [x] **PARSE-05**: App parser config.json korrekt (workflow-indstillinger)

### UI/UX

- [x] **UI-01**: App følger macOS system theme (dark/light mode automatisk)
- [x] **UI-02**: App bruger NavigationSplitView med sidebar → content → detail layout
- [x] **UI-03**: App har et rent, professionelt design der føles som en native Mac-app

- **ADV-04**: Multi-window support

## Out of Scope

| Feature | Reason |
|---------|--------|
| In-app file editing | Konflikter med editor-of-choice workflow. "Open in Cursor" er simplere |
| Custom themes/farver | System theme er tilstrækkeligt. Undgår settings-kompleksitet |
| Real-time collaboration | GSD er filbaseret, ikke database. Ingen conflict resolution |
| Git integration | Scope creep. Git GUIs eksisterer allerede (Tower, Fork) |
| Gantt chart tidslinje | GSD-faser er mål-baserede, ikke dato-baserede |
| Customizable dashboard widgets | Over-engineering for read-only monitoring |
| Cloud sync | Lokale filer kun. Ingen server-dependency |
| gsd-console som dependency | Bygger alt selv — ingen Bun/Node runtime |
