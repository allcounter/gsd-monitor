# Requirements: GSD Monitor

**Defined:** 2026-02-17
**Core Value:** Et live, grafisk overblik over alle GSD-projekter på tværs af ~/Developer — så du altid ved hvor du er, uden at åbne en terminal.

## v1.2 Requirements

Requirements for v1.2 GSD Command Runner. Each maps to roadmap phases.

### Process Execution

- [x] **PROC-01**: User can run a GSD command from the app via embedded command runner
- [x] **PROC-02**: App resolves claude CLI path from user's shell environment (PATH augmentation)
- [x] **PROC-03**: App spawns process with PTY for ANSI color support
- [x] **PROC-04**: User sees live-streamed output as command runs
- [x] **PROC-05**: User can cancel a running command with process group kill

### Output Display

- [x] **OUTP-01**: User sees raw terminal output in a scrollable panel with auto-scroll
- [x] **OUTP-02**: User sees stderr lines colored distinctly from stdout
- [x] **OUTP-03**: User sees exit code when command completes
- [x] **OUTP-04**: User can toggle between structured view and raw output
- [x] **OUTP-05**: Structured view parses GSD banners, progress, and task status
- [x] **OUTP-06**: Output panel uses rolling buffer (5000 lines) to prevent memory growth

### Command Triggering

- [x] **TRIG-01**: User can trigger GSD commands via context buttons on phases and plans
- [x] **TRIG-02**: User sees running state indicator when a command is active
- [x] **TRIG-03**: User can trigger GSD commands via Cmd+K command palette
- [x] **TRIG-04**: User can view and re-run previous commands from command history

### Integration & Safety

- [x] **SAFE-01**: Only one command runs per project at a time (queue model)
- [x] **SAFE-02**: FSEvents project reload suppressed during active command execution
- [x] **SAFE-03**: User sees confirmation dialog before cancelling a running command
- [x] **SAFE-04**: User sees actionable error message with recovery suggestions on command failure

## Future Requirements

### Manual Editor Addition

- **EDIT-01**: User can manually add editors not auto-detected in /Applications

## Out of Scope

| Feature | Reason |
|---------|--------|
| Full general terminal emulator | Scope mismatch — only GSD commands, not a shell |
| Interactive stdin prompts | GSD commands run non-interactively; interactive mode deferred |
| Multiple concurrent commands per project | GSD agents mutate same .planning/ files — would corrupt state |
| Remote/SSH command execution | Local-only developer utility |
| iOS/iPad version | macOS only |
| Cloud sync | Local files only |
