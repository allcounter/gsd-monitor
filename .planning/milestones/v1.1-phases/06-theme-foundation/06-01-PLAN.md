**Accent colors (normal):** red (#cc241d), green (#98971a), yellow (#d79921), blue (#458588), purple (#b16286), aqua (#689d6a), orange (#d65d0e), gray (#928374)

**Accent colors (bright):** brightRed (#fb4934), brightGreen (#b8bb26), brightYellow (#fabd2f), brightBlue (#83a598), brightPurple (#d3869b), brightAqua (#8ec07c), brightOrange (#fe8019)

3. Semantic aliases as static properties on the same Theme enum:
```swift
// Status colors (per user decision)
```

Do NOT use Asset Catalog — use code-based Color extension per user decision (two naming layers: raw Gruvbox + semantic).

1. In AppDelegate.swift, add to applicationDidFinishLaunching:
```swift
NSApp.appearance = NSAppearance(named: .darkAqua)
```
This forces dark mode regardless of system appearance setting — per user decision, use NSApp.appearance (not preferredColorScheme).

2. In GSDMonitorApp.swift, add window styling to the WindowGroup for immersive title bar:
- Add `.windowStyle(.hiddenTitleBar)` to WindowGroup if available, OR
- Use `.toolbar { ToolbarItem(placement: .automatic) { Color.clear } }` approach
- Apply `.containerBackground(Theme.bg0, for: .window)` if targeting macOS 15+
- The goal is title bar blending seamlessly with bg0 background — per user decision: "immersive look, not native title bar"

```swift

```

Note: The window background color should be set to Theme.bg0. Apply `.background(Theme.bg0)` to ContentView's NavigationSplitView if needed.
