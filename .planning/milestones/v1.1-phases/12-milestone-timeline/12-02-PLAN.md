**B. MilestoneTimelineView.swift** — top-level timeline:

Props: `project: Project`, `projectName: String = ""`

State: `@SwiftUI.State private var expandedMilestones: Set<String> = []`

Computed: `milestoneGroups` from `project.roadmap?.milestones ?? []`

Helper: `phasesForMilestone(_ milestone: Milestone) -> [Phase]` — filters `project.roadmap?.phases` where `milestone.phaseNumbers.contains(phase.number)`, sorted by number.

Helper: `scrollTargetID` — name of first non-complete milestone (current milestone for auto-scroll). If all complete, nil.

Layout:
```swift
ScrollViewReader { proxy in
    ScrollView(.vertical, showsIndicators: false) {
        VStack(alignment: .leading, spacing: 0) {
            ForEach(milestoneGroups) { group in
                MilestoneGroupView(
                    milestone: group,
                    phases: phasesForMilestone(group),
                    project: project,
                    isExpanded: expandedMilestones.contains(group.name),
                    onToggle: { toggleExpansion(group.name) }
                )
                .id(group.name)
            }
        }
    }
    .onAppear {
        if let target = scrollTargetID {
            DispatchQueue.main.async {
                proxy.scrollTo(target, anchor: .top)
            }
        }
    }
}
```

Toggle function: `withAnimation(.easeInOut(duration: 0.25)) { expandedMilestones.formSymmetricDifference([name]) }`

Guard: if milestones empty, render EmptyView.

**C. DetailView.swift integration:**

Add `MilestoneTimelineView` in the pinned header section, AFTER `StatsGridView(project: project)` and BEFORE the `Divider()`. Wrap in a conditional:

```swift
if let roadmap = project.roadmap, !roadmap.milestones.isEmpty {
    MilestoneTimelineView(project: project, projectName: projectName)
        .frame(maxHeight: 220)
}
```

**Layout note:** MilestoneTimelineView is placed in the pinned header. It contains its own internal ScrollView (inside MilestoneTimelineView) for auto-scroll functionality. The `.frame(maxHeight: 220)` on the caller constrains the visible height. The internal ScrollView enables `ScrollViewReader.proxy.scrollTo` to work correctly, scrolling to the current milestone on load. This does NOT conflict with the outer phase-cards ScrollView because it is a separate, contained scroll region.
