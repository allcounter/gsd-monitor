  <name>Task 1: Create CircularProgressRing component</name>
  <files>GSDMonitor/Views/Components/CircularProgressRing.swift</files>
  <action>
Create a new SwiftUI view `CircularProgressRing` in the Components directory.

Parameters:
- `progress: Double` (0.0 to 1.0)
- `ringColor: Color` — foreground ring color
- `backgroundColor: Color` — background ring color (default: `Theme.bg2`)
- `lineWidth: CGFloat` — stroke width

Animation strategy (CRITICAL — prevents FSEvents re-animation):
- Use `@State private var hasAppeared = false` as animation trigger
- In body: `Circle().trim(from: 0, to: hasAppeared ? progress : 0)`
- Use `.animation(.easeOut(duration: 0.6), value: hasAppeared)` — animate the FLAG, not progress
- Set `hasAppeared = true` in `.onAppear { }` — fires once per view lifecycle
- Apply `.rotationEffect(.degrees(-90))` so ring starts at 12 o'clock (top)

Structure:
```swift
struct CircularProgressRing: View {
    let progress: Double
    let ringColor: Color
    var backgroundColor: Color = Theme.bg2
    let lineWidth: CGFloat

    @State private var hasAppeared = false

    var body: some View {
        Circle()
            .stroke(backgroundColor, lineWidth: lineWidth)
            .overlay {
                Circle()
                    .trim(from: 0, to: hasAppeared ? progress : 0)
                    .stroke(ringColor, style: StrokeStyle(lineWidth: lineWidth, lineCap: .round))
            }
            .rotationEffect(.degrees(-90))
            .animation(.easeOut(duration: 0.6), value: hasAppeared)
            .onAppear {
                hasAppeared = true
            }
    }
}
```

Do NOT use `.animation(_:value: progress)` — this would cause re-animation on every FSEvents data reload.
Do NOT use implicit `.animation()` without value parameter — deprecated in SwiftUI 3.0.
