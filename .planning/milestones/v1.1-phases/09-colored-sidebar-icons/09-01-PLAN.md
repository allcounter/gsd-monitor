  <action>
In the private `ProjectRow` struct in SidebarView.swift, add:

1. A `ProjectStatus` enum (private to the file) with cases: `.notStarted`, `.active`, `.complete`

2. A computed property `projectStatus` on ProjectRow that derives status from `project.roadmap`:
   - No roadmap -> `.notStarted`
   - All phases done (non-empty) -> `.complete`
   - Any phase `.inProgress` -> `.active`
   - Otherwise -> `.notStarted`
   This mirrors the existing `matchesStatusFilter` logic at lines 54-67.

3. A computed property `statusColor` returning:
   - `.notStarted` -> `Theme.statusNotStarted` (fg4 gray)
   - `.active` -> `Theme.statusActive` (yellow)
   - `.complete` -> `Theme.statusComplete` (green)

4. A computed property `statusSymbol` returning SF Symbol name:
   - `.notStarted` -> `"folder"`
   - `.active` -> `"folder.fill"`
   - `.complete` -> `"checkmark.circle.fill"`

5. In the ProjectRow `body`, add the icon to the HStack at line 152 (the one containing project name and phase count), BEFORE the `Text(project.name)`:

```swift
Image(systemName: statusSymbol)
    .symbolRenderingMode(.hierarchical)
    .foregroundStyle(statusColor)
    .font(.system(size: 14))
```

Use `.font(.system(size: 14))` for sizing (NOT `.frame()` — SF Symbols scale via font, not frame).
Use `.symbolRenderingMode(.hierarchical)` for depth.
Use `.foregroundStyle()` (not `.foregroundColor()`) per codebase convention.

Do NOT create a separate status field on the model — derive from phases (avoids state duplication).
Do NOT use `.palette` rendering mode (single-color icon, not multi-layer).
