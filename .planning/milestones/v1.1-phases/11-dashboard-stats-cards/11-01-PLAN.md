**State.swift changes:**

Add corresponding CodingKeys:

**StateParser.swift changes:**

1. Add two new properties to `StateWalker`:
   - `var totalExecutionTime: String?`
   - `var currentMilestone: String?`
   - `private var inVelocitySection = false`

2. In `visitParagraph(_:)`, add parsing for the "Current focus:" line:
   - The STATE.md line looks like: `**Current focus:** Phase 10 - Enhanced Empty States (v1.1 Visual Overhaul) — Complete`
   - The `plainText` version will be: `Current focus: Phase 10 - Enhanced Empty States (v1.1 Visual Overhaul) — Complete`
   - Check if `text.starts(with: "Current focus:")` (plainText strips bold markers)
   - Extract the milestone name from parentheses using regex `\(([^)]+)\)` — capture group gives "v1.1 Visual Overhaul"
   - Set `currentMilestone` to the captured group

3. In `visitParagraph(_:)`, detect the Velocity section:
   - Check if `text` contains "Velocity:" — set `inVelocitySection = true`

4. In `visitHeading(_:)`, reset `inVelocitySection = false` when entering a new heading (similar to how `inDecisionsSection` is reset on heading level <= 2).

5. In `visitListItem(_:)`, add parsing for "Total execution time:" when `inVelocitySection` is true:
   - Use `listItem.plainText` (not `format()` which includes markdown)
   - Check if text contains "Total execution time:"
   - Extract everything after "Total execution time:" and trim whitespace
   - Set `totalExecutionTime` to the extracted value (e.g. "~1.8 hours")
   - Process this BEFORE the existing decisions/blockers logic (use early return or separate if-block)

6. Update `StateParser.parse()` return to pass the two new fields:
   ```swift
   return State(
       currentPhase: walker.currentPhase,
       currentPlan: walker.currentPlan,
       status: walker.status ?? "",
       lastActivity: walker.lastActivity,
       decisions: walker.decisions,
       blockers: walker.blockers,
       totalExecutionTime: walker.totalExecutionTime,
       currentMilestone: walker.currentMilestone
   )
   ```

**Important:** Use `plainText` (not `format()`) for the "Current focus:" check in `visitParagraph`, since bold markers `**` are stripped by plainText. For the list item, use `plainText` as well for cleaner extraction.
