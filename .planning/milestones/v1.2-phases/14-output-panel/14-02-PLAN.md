**OutputPanelView.swift — wire failure notification:**
- Access NotificationService from the environment or from a new `@Environment` property. NOTE: NotificationService is currently created in ContentView.task closure and stored in `@SwiftUI.State private var notificationService: NotificationService?`. It is NOT currently in the environment. Two options:
  - Option A (preferred): Add `notificationService` to the environment in ContentView alongside commandRunnerService, then access via `@Environment(NotificationService.self)` in OutputPanelView. This requires making NotificationService non-optional in ContentView.
  - Option B: Pass NotificationService as a parameter to OutputPanelView.
  - Choose Option A: In ContentView, change `notificationService` to `@SwiftUI.State private var notificationService = NotificationService(projectService: projectService)` — but this has a chicken-and-egg problem since projectService is also @State. Instead, create NotificationService in `.task` as before but inject it via `.environment()` after creation. The simplest approach: create it eagerly alongside projectService. But NotificationService requires projectService in its init.
  - **Simplest correct approach:** Keep NotificationService creation in `.task` but also add `.environment(notificationService)` conditionally. OR just pass it through. Actually the cleanest: add an `@Environment(NotificationService?.self)` and make the environment optional. OR simpler yet: have OutputPanelView accept an optional NotificationService parameter and pass it from DetailView which gets it from ContentView. Actually that threads it through too many layers.
  - **Final approach:** Have OutputPanelView handle its own notification directly using `UNUserNotificationCenter` without going through NotificationService. Add a private method `sendFailureNotification(projectName:command:exitCode:)` directly in OutputPanelView that creates and submits the notification content. This avoids the injection complexity and keeps the feature self-contained. NotificationService's `requestPermissionIfNeeded` can be called independently since `UNUserNotificationCenter.current()` is a singleton.
- Add `.onChange(of: currentRun?.state)` observer in OutputPanelView:
  - When state transitions to `.failed` or `.crashed`, and `currentRun` exists:
    - Call the local `sendFailureNotification` method with `projectName: project.name`, `command: currentRun.command`, `exitCode: currentRun.exitCode ?? -1`
  - Do NOT send notification for `.succeeded` or `.cancelled` (per user decision: notifications only on failure)

**Private notification method in OutputPanelView:**
```swift
private func sendFailureNotification(projectName: String, command: String, exitCode: Int32) {
    _Concurrency.Task {
        let center = UNUserNotificationCenter.current()
        let settings = await center.notificationSettings()
        guard settings.authorizationStatus == .authorized || settings.authorizationStatus == .provisional else { return }

        let content = UNMutableNotificationContent()
        content.title = "Command Failed — \(projectName)"
        content.body = "Exit code \(exitCode): \(command)"
        content.sound = .default
        content.interruptionLevel = .timeSensitive

        let request = UNNotificationRequest(
            identifier: UUID().uuidString,
            content: content,
            trigger: nil
        )
        try? await center.add(request)
    }
}
```
