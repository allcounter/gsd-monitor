  <action>
Create two new view files in a new `GSDMonitor/Views/OutputPanel/` directory.

**OutputPanelView.swift:**
- `import SwiftUI`
- `struct OutputPanelView: View` accepting `project: Project` as a parameter
- Access `CommandRunnerService` via `@Environment(CommandRunnerService.self) private var commandRunner`
- Panel header: `HStack` with `Text("Output")` in `.font(.headline)` + `Theme.textPrimary`. Add `Spacer()`. This header will be extended in Plan 02 with a segmented toggle.
- Header background: `Theme.bg1` with horizontal padding 12, vertical padding 8
- `Divider().background(Theme.bg2)` below header
- Body: render `OutputRawView` passing the displayed lines (will be extended to toggle in Plan 02)
- Compute `currentRun`: check `commandRunner.activeRuns[project.path.path]` first, then fall back to `commandRunner.recentRuns.last(where: { $0.projectPath.path == project.path.path })`
- Rolling buffer: computed property `displayedLines` — if `currentRun?.outputLines.count ?? 0 > 5000`, return `Array(lines.suffix(5000))`, otherwise return lines directly. Cache in `@SwiftUI.State var cachedLines: [CommandRun.OutputLine] = []` updated via `.onChange(of: currentRun?.outputLines.count)` to avoid recomputing on every render.
- If no `currentRun`, show `ContentUnavailableView` with "No commands run yet" message using `Theme.textMuted` and `Theme.fg4`, matching the empty state pattern in DetailView.
- Wrap entire body in `VStack(spacing: 0)`
- Background: `Theme.bg0`

**OutputRawView.swift:**
- `import SwiftUI`
- `struct OutputRawView: View` accepting `let lines: [CommandRun.OutputLine]`
- Use `ScrollViewReader { proxy in ScrollView(.vertical) { ... } }` pattern
- Inside ScrollView: `LazyVStack(alignment: .leading, spacing: 2)` with `ForEach(Array(lines.enumerated()), id: \.offset)` rendering each line
- Each line: `Text(line.text)` with `.font(.system(.body, design: .default))` (system font per user decision — NOT monospace), `.foregroundStyle(line.stream == .stderr ? Theme.brightRed : Theme.fg1)`, `.textSelection(.enabled)`, `.id(index)`
- Bottom sentinel: `Color.clear.frame(height: 1).id("bottom")`
- Auto-scroll: `@SwiftUI.State private var isAtBottom = true`. On `.onChange(of: lines.count)`, if `isAtBottom`, call `proxy.scrollTo("bottom", anchor: .bottom)`. Start simple — no preference key offset tracking for now (per research "simpler alternative for MVP" recommendation).
- Padding: `.padding(.horizontal, 12).padding(.vertical, 8)` on the LazyVStack

**CRITICAL patterns:**
- Use `_Concurrency.Task` for any async Task blocks (GSDMonitor.Task shadows Swift Task)
- Register both new files in project.pbxproj: PBXBuildFile, PBXFileReference, create OutputPanel group under Views group, add to Sources build phase. Follow the established deterministic ID pattern from Phase 13 (use OUTP1401 prefix for IDs).
