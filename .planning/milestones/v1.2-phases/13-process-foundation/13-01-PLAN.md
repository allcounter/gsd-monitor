  <action>
**ShellEnvironmentService.swift:**
- Top-level `struct ShellEnvironmentService: Sendable` (nonisolated, no actor needed â€” called from CommandRunnerService)
- `func captureLoginShellEnvironment() async throws -> [String: String]`:
  - Spawn `/bin/zsh` with arguments `["-l", "-c", "env"]`
  - Parse output as KEY=VALUE lines (value may contain `=`, use `split(separator: "=", maxSplits: 1)`)
  - Return dictionary of environment variables
  - Per research: do NOT set process.environment â€” let zsh inherit and load .zprofile/.zshrc
  - Discard stderr (assign separate Pipe to standardError)
- `func resolveClaudePath() async -> URL?`:
  - First: capture shell environment, search PATH for `claude` executable using `FileManager.default.isExecutableFile(atPath:)`
  - Fallback: check known locations: `/usr/local/bin/claude`, `/opt/homebrew/bin/claude`, `~/.local/bin/claude`, `~/.claude/local/claude`
  - Return first found executable URL, or nil
  - Per research pitfall 4: re-capture environment each time (not cached) â€” it's fast (<200ms) and always current
- `enum ShellEnvironmentError: Error, LocalizedError` with cases: `parseFailure`, `claudeNotFound`
  - Provide `errorDescription` per user decision: claude not found case should guide with "Install via: npm install -g @anthropic-ai/claude-code"

**GSDOutputParser.swift:**
- `struct GSDOutputParser: Sendable`
- `func parse(line: String) -> GSDLineInfo` where `GSDLineInfo` has optional fields: `phaseNumber: Int?`, `planNumber: Int?`, `isTaskComplete: Bool`, `isError: Bool`
- Use Swift Regex (available in Swift 5.7+/macOS 13+):
  - Phase: `/##?\s+Phase\s+(\d+)/`
  - Plan: `/[Pp]lan\s+(\d+)/`
  - Task complete: check for "completed", "DONE", or checkmark characters
  - Error: check for "ERROR:", "FAILED:", or cross-mark characters
- `func aggregate(lines: [GSDLineInfo]) -> GSDMetadata` â€” reduces line-by-line info into a single GSDMetadata summary
- Per research: LOW confidence on patterns â€” treat parsed metadata as best-effort, document that patterns need iteration

Register both files in project.pbxproj (ShellEnvironmentService in Services group, GSDOutputParser in Utilities group).
