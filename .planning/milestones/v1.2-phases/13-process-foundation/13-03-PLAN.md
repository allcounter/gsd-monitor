Create `actor CommandHistoryStore` in Services/:

**Purpose:** Actor-isolated JSON file persistence. Actor isolation prevents file contention when multiple projects complete commands simultaneously (research open question 5).

**Properties:**
- `private let fileURL: URL` — `~/Library/Application Support/GSDMonitor/command-history.json`
- `private let maxRunsPerProject: Int = 200` (research recommendation: count-based, 200 per project)

**Init:**
- Compute fileURL from `FileManager.default.urls(for: .applicationSupportDirectory, in: .userDomainMask).first!`
- Append `"GSDMonitor"` directory component (create with `withIntermediateDirectories: true`)
- Append `"command-history.json"` filename

**Methods:**
- `func loadAll() throws -> [CommandRun]` — read JSON file, decode `[CommandRun]`, return empty array if file doesn't exist
- `func save(_ run: CommandRun) async throws` — load existing, append new run, trim per project to `maxRunsPerProject` (keep newest), encode, write atomically
- `func loadForProject(_ projectPath: URL) throws -> [CommandRun]` — load all, filter by projectPath
- `func removeQueuedCommand(id: UUID) async throws` — load, remove matching id (only if state == .queued), save back — per user decision: queued commands individually removable

**Trim logic:** Group by `projectPath.path`, for each group sort by `startTime` descending, take first `maxRunsPerProject`, flatten back. Per research: 200 runs keeps JSON < 5 MB.

**Encoding:** Use `JSONEncoder()` with `.prettyPrinted` output formatting and `.iso8601` date strategy. Use `JSONDecoder()` with `.iso8601` date strategy. Write with `Data.write(to:options:.atomic)` to prevent corruption on crash.

Register in project.pbxproj (Services group).

<task type="auto">
  <name>Task 2: Create CommandRunnerService with per-project queues</name>
  <files>GSDMonitor/Services/CommandRunnerService.swift</files>
  <action>
Create `@MainActor @Observable final class CommandRunnerService` in Services/:

**Observable state (for Phase 14 UI binding):**
- `var activeRuns: [String: CommandRun] = [:]` — keyed by project path, the currently running command per project
- `var projectQueues: [String: [CommandRequest]] = [:]` — keyed by project path, visible queued commands per project
- `var recentRuns: [CommandRun] = []` — last few completed runs (in-memory, for quick UI access)

**Private state:**
- `private var queueStreams: [String: (stream: AsyncStream<CommandRequest>, continuation: AsyncStream<CommandRequest>.Continuation)] = [:]`
- `private var queueTasks: [String: Task<Void, Never>] = [:]`
- `private var activeProcessActors: [String: ProcessActor] = [:]` — for cancel access
- `private let shellEnvService = ShellEnvironmentService()`
- `private let historyStore = CommandHistoryStore()`
- `private let outputParser = GSDOutputParser()`

**Public API:**

`func enqueue(_ request: CommandRequest)`:
- Add request to `projectQueues[request.projectPath.path]` (visible queue)
- Get or create AsyncStream queue for this project path
- Yield request to continuation
- If no consumer task exists for this project, start one via `startConsuming(projectPath:stream:)`

`func cancelRunningCommand(forProject projectPath: URL) async`:
- Per user decision: cancellation requires user confirmation — but confirmation is UI (Phase 14). This method is the "execute cancel" action called after confirmation.
- Guard `activeProcessActors[projectPath.path]` exists
- Call `await actor.cancel()`
- Update `activeRuns[projectPath.path]?.state = .cancelled`

`func removeQueuedCommand(id: UUID, forProject projectPath: URL) async`:
- Remove from `projectQueues[projectPath.path]` by id
- Remove from `historyStore` via `removeQueuedCommand(id:)`
- Per user decision: queued commands individually removable

`func loadHistory(forProject projectPath: URL) async -> [CommandRun]`:
- Load from historyStore, return for UI display
- Per user decision: history persists across restarts

**Private methods:**

`private func getOrCreateQueue(for projectPath: String) -> (stream: AsyncStream<CommandRequest>, continuation: AsyncStream<CommandRequest>.Continuation)`:
- If queue exists, return it
- Otherwise create new `AsyncStream.makeStream(of: CommandRequest.self)` (Swift 5.9+ API)
- Store and start consumer

`private func startConsuming(projectPath: String, stream: AsyncStream<CommandRequest>)`:
- Create `Task` that `for await request in stream`:
  - Remove request from visible `projectQueues`
  - Call `await executeCommand(request)`
  - Loop continues to next queued command (FIFO serialization per project)

`private func executeCommand(_ request: CommandRequest) async`:
1. Resolve claude path: `guard let claudeURL = await shellEnvService.resolveClaudePath()` else → create failed CommandRun with ShellEnvironmentError.claudeNotFound, store, return
2. Capture full environment: `let env = try await shellEnvService.captureLoginShellEnvironment()`
3. Create CommandRun with state .running, store in `activeRuns[projectPath]`
4. Create ProcessActor, store in `activeProcessActors[projectPath]`
5. Call `let outputStream = await actor.run(executableURL: claudeURL, arguments: request.arguments, environment: env, workingDirectory: request.projectPath)`
6. Collect output: `for await line in outputStream` — append to `run.outputLines`, update `activeRuns` (triggering @Observable updates)
7. On stream completion: determine final state from process exit code (0 = succeeded, non-zero = failed)
8. Parse metadata: `let metadata = outputParser.aggregate(lines: lineInfos)`
9. Update CommandRun with endTime, exitCode, metadata, final state
10. **Auto-retry logic (per user decision):** If state == .failed AND !run.autoRetried → create new CommandRequest, set autoRetried=true on retry run, enqueue it. If already auto-retried → stop, user must manually re-trigger.
11. **Crashed state detection:** If `process.terminationReason == .uncaughtSignal` AND not cancelled → set state to .crashed (per user decision: distinct from normal failure)
12. Persist to history: `try await historyStore.save(run)`
13. Add to `recentRuns` (keep last 20 in memory)
14. Clean up: remove from `activeRuns`, remove from `activeProcessActors`

**Re-run support (per user decision: one-click re-run from history):**
`func rerun(_ run: CommandRun)`:
- Create CommandRequest from the run's command/arguments/projectPath/projectName
- Call `enqueue(request)`

Register in project.pbxproj (Services group).
