  <action>
Create `actor ProcessActor` in Services/:

**Properties:**
- `private var process: Process?`
- `private var ptyMasterFD: Int32 = -1` — PTY master file descriptor (keep alive to prevent SIGHUP)
- `private var continuation: AsyncStream<CommandRun.OutputLine>.Continuation?`
- `private var isCancelling: Bool = false`

**Core method — `func run(...) -> AsyncStream<CommandRun.OutputLine>`:**
- Parameters: `executableURL: URL`, `arguments: [String]`, `environment: [String: String]`, `workingDirectory: URL`
- Returns `AsyncStream<CommandRun.OutputLine>`
- Implementation:
  1. **PTY setup:** Call `Darwin.openpty(&masterFD, &slaveFD, nil, nil, nil)`. Store masterFD in self.ptyMasterFD. Create `FileHandle(fileDescriptor: slaveFD, closeOnDealloc: true)` for process stdin. Do NOT use forkpty() — unsafe from Swift.
  2. **Pipes:** Create `Pipe()` for stdout and stderr separately. These are NOT on the PTY — stdout/stderr go through Pipes for clean readabilityHandler streaming.
  3. **Process configuration:**
     - `process.executableURL = executableURL`
     - `process.arguments = arguments`
     - `process.environment = environment` (the full shell environment from ShellEnvironmentService)
     - `process.currentDirectoryURL = workingDirectory`
     - `process.standardInput = slaveHandle` (PTY slave)
     - `process.standardOutput = stdoutPipe`
     - `process.standardError = stderrPipe`
  4. **readabilityHandler bridge (Swift 6 safe):**
     - For both stdout and stderr pipes: set `readabilityHandler` closure that calls `fh.availableData`, checks non-empty, converts to String (UTF-8), and yields `CommandRun.OutputLine` to the continuation.
     - Per research: readabilityHandler runs on GCD thread. Do NOT directly mutate actor state. Yield directly to continuation (continuation.yield is thread-safe per AsyncStream docs).
  5. **terminationHandler (authoritative completion):**
     - Set `process.terminationHandler` that:
       a. Nullifies both readabilityHandlers (prevent further callbacks)
       b. Does a final `readDataToEndOfFile()` on both pipes to flush remaining data, yielding any non-empty results
       c. Closes PTY master FD via `Darwin.close(masterFD)`
       d. Calls `continuation.finish()`
     - Per research: Do NOT rely on readabilityHandler empty-data for EOF — terminationHandler is authoritative.
  6. **Launch:** `try process.run()` (not deprecated `launch()`). On error, close PTY master, finish continuation.
- Create the AsyncStream with `AsyncStream<CommandRun.OutputLine> { continuation in ... }` pattern. Store continuation for cancel method access.

**Cancel method — `func cancel() async`:**
- Guard process exists and is running. Set `isCancelling = true`.
- Step 1: `process.interrupt()` — sends SIGINT (correct for claude CLI, simulates Ctrl+C)
- Step 2: Poll loop — check `process.isRunning` every 200ms for up to 4 seconds (research recommendation)
- Step 3: If still running after 4s, force kill:
  - `let pid = process.processIdentifier`
  - `Darwin.kill(pid, SIGKILL)` — kill direct child
  - `Darwin.kill(-pid, SIGKILL)` — attempt process group kill (may be no-op if PGID != pid, but safe to try)
- Per research process group caveat: Foundation.Process may not set new PGID. The kill(-pid) is best-effort. For claude's use case, SIGINT to the main process is usually sufficient since claude handles its own cleanup.

**Helper — `func isRunning() -> Bool`:**
- Returns `process?.isRunning ?? false`

**Helper — `func processIdentifier() -> Int32?`:**
- Returns `process?.processIdentifier`

**Error enum:**
- `enum ProcessError: Error, LocalizedError` with cases: `ptyCreationFailed`, `processLaunchFailed(Error)`, `alreadyRunning`
- Provide `errorDescription` for each

**Critical implementation notes:**
- Import `Darwin` at top for openpty/kill
- The continuation is yielded to directly from readabilityHandler (not through actor method) because `AsyncStream.Continuation.yield` is explicitly documented as thread-safe
- PTY master MUST stay open until terminationHandler fires — closing it sends SIGHUP to child
- The slaveFD FileHandle uses `closeOnDealloc: true` so it closes automatically when process releases it
