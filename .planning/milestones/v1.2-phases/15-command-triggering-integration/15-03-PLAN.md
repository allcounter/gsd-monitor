Change `private let maxRunsPerProject: Int = 200` to `private let maxRunsPerProject: Int = 50`. This is a single-line change — per user decision, retain last 50 command runs per project.

**ProjectService.swift:**

1. Modify `startMonitoring()` to accept a `CommandRunnerService` parameter:
```swift
func startMonitoring(commandRunner: CommandRunnerService? = nil) {
```
The `nil` default preserves backward compatibility if called without it (though all call sites should be updated).

2. Inside the `for await changedURLs in eventStream.debounce(for: .seconds(1))` loop, before calling `await reloadProject(at:)`, add the SAFE-02 suppression check:
```swift
for projectPath in affectedProjects {
    // SAFE-02: Suppress reload while a command is actively writing to .planning/
    if let runner = commandRunner, runner.activeRuns[projectPath] != nil {
        continue  // Skip — FSEvents will fire again after command completes
    }
    await reloadProject(at: URL(fileURLWithPath: projectPath))
}
```

This works because: when the command finishes, `activeRuns[key]` is cleared, and the CLI's final file writes trigger another FSEvents event that passes through the suppression gate normally. No explicit "reload on completion" hook is needed.

3. Do NOT double-debounce (per research pitfall 4). The existing 1-second debounce is sufficient.

**ContentView.swift:**

1. Update the `projectService.loadProjects()` call to pass `commandRunnerService` down. Since `loadProjects()` calls `startMonitoring()` internally, we need to make `commandRunnerService` available to `ProjectService`.

   The cleanest approach: store the `CommandRunnerService` reference in `ProjectService` so `startMonitoring()` can use it without changing `loadProjects()` signature.

   Alternative (simpler): Add a property to `ProjectService`:
   ```swift
   var commandRunner: CommandRunnerService?
   ```
   Set it in `ContentView.task` before calling `loadProjects()`:
   ```swift
   projectService.commandRunner = commandRunnerService
   await projectService.loadProjects()
   ```

   Then in `ProjectService.startMonitoring()`, use `self.commandRunner` directly instead of a parameter:
   ```swift
   func startMonitoring() {
       // ... existing setup ...
       monitoringTask = _Concurrency.Task {
           for await changedURLs in eventStream.debounce(for: .seconds(1)) {
               // ... existing affected projects logic ...
               for projectPath in affectedProjects {
                   if let runner = commandRunner, runner.activeRuns[projectPath] != nil {
                       continue
                   }
                   await reloadProject(at: URL(fileURLWithPath: projectPath))
               }
           }
       }
   }
   ```

   This avoids changing the `startMonitoring()` signature and breaking `addProjectManually()`/`removeManualProject()` call sites.

2. In `ContentView.task`, add `projectService.commandRunner = commandRunnerService` before the `await projectService.loadProjects()` call.
