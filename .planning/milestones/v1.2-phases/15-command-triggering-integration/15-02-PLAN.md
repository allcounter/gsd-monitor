**2. PaletteCommand struct** (at file scope):
```swift
struct PaletteCommand: Identifiable {
    let id: String
    let name: String
    let icon: String           // SF Symbol name
    let slash: String?         // GSD slash command (nil for app actions)
    let needsProject: Bool     // true for GSD commands
    let needsPhaseNumber: Bool // true for plan-phase, execute-phase, verify, discuss
    let appAction: (() -> Void)?  // non-nil for app actions

    static let allCommands: [PaletteCommand] = [
        // GSD commands
        PaletteCommand(id: "discuss", name: "Discuss Phase", icon: "bubble.left.and.bubble.right", slash: "/gsd:discuss-phase", needsProject: true, needsPhaseNumber: true, appAction: nil),
        PaletteCommand(id: "plan", name: "Plan Phase", icon: "list.bullet.clipboard", slash: "/gsd:plan-phase", needsProject: true, needsPhaseNumber: true, appAction: nil),
        PaletteCommand(id: "execute", name: "Execute Phase", icon: "play.fill", slash: "/gsd:execute-phase", needsProject: true, needsPhaseNumber: true, appAction: nil),
        PaletteCommand(id: "verify", name: "Verify Work", icon: "checkmark.seal", slash: "/gsd:verify-work", needsProject: true, needsPhaseNumber: true, appAction: nil),
        PaletteCommand(id: "research", name: "Research Phase", icon: "magnifyingglass", slash: "/gsd:research-phase", needsProject: true, needsPhaseNumber: true, appAction: nil),
        // App actions (no project/params needed)
        PaletteCommand(id: "refresh", name: "Refresh Projects", icon: "arrow.clockwise", slash: nil, needsProject: false, needsPhaseNumber: false, appAction: nil),
    ]
}
```

Note: The `appAction` closure for "Refresh Projects" will be wired in the view (calls `projectService.loadProjects()`). Store `appAction` as `nil` in the static list and handle it in the view's action dispatch.

**3. CommandPaletteView struct:**

Properties:
- `let projectService: ProjectService`
- `let onSelectProject: (UUID) -> Void`   // keep for backward compat with ContentView
- `@Environment(\.dismiss) private var dismiss`
- `@Environment(CommandRunnerService.self) private var commandRunner`
- `@SwiftUI.State private var step: PaletteStep = .selectCommand`
- `@SwiftUI.State private var query = ""`
- `@FocusState private var searchFocused: Bool`

Body: A VStack(spacing: 0) with:
- A breadcrumb/back bar at top (only visible when step != .selectCommand): shows current step label + back button
- The search field (same style as current — magnifying glass + TextField + clear button)
- A Divider
- The results list (changes content based on step)

Frame: `.frame(width: 600, height: 400)`

On `.onAppear`:
- `step = .selectCommand` (reset on every open — prevents stale state per research pitfall 3)
- `query = ""`
- `searchFocused = true`

**Step-by-step flow:**

**`.selectCommand`:** Show filtered list of `PaletteCommand.allCommands` matching `query` (filter by `name.localizedCaseInsensitiveContains(query)`, show all if query is empty). Each row: icon + name. On tap:
- If `!command.needsProject`: execute app action immediately (for "Refresh Projects": `Task { await projectService.loadProjects() }`), then `dismiss()`.
- If `command.needsProject`: set `step = .selectProject(command)`, clear `query`.

**`.selectProject(command)`:** Show filtered list of `projectService.projects` matching `query` (filter by `name`). Each row: project name + path subtitle. On tap:
- If `command.needsPhaseNumber`: set `step = .promptPhaseNumber(command, project)`, clear `query`.
- Else: enqueue command with no extra args, dismiss.

**`.promptPhaseNumber(command, project)`:** Show list of phases from `project.roadmap?.phases ?? []`. Each row: "Phase {number}: {name}" + status badge. On tap:
- Build arguments: `[command.slash!, "\(phase.number)"]`
- Enqueue: `commandRunner.enqueue(CommandRequest(command: "claude", arguments: args, projectPath: project.path, projectName: project.name))`
- Also call `onSelectProject(project.id)` so the sidebar selects this project
- `dismiss()`

**Back navigation:** The breadcrumb bar shows a back button (chevron.left). Tapping back:
- From `.selectProject` → `.selectCommand`
- From `.promptPhaseNumber` → `.selectProject(command)`
- Clear query on each back navigation.

**Keyboard:** Escape key already handled by the sheet's cancel action. Enter key could select first item (optional, Claude's discretion on keyboard nav details).

**Styling:**
- Use `Theme.bg0` for background, `Theme.surface` for search field background
- Use `Theme.textPrimary` for item text, `Theme.textSecondary` for subtitles
- Use `Theme.accent` for the breadcrumb text showing current step
- List rows use `.listRowBackground(Theme.bg0)` for consistency
