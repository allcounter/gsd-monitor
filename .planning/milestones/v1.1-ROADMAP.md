### 🚧 v1.1 Visual Overhaul (In Progress)

**Milestone Goal:** Gruvbox Dark farvesystem og grafiske forbedringer der giver appen en unik, poleret visuel identitet.

#### Phase 6: Theme Foundation
**Goal**: Gruvbox Dark farvepalet integreret som centraliseret tema-system
**Depends on**: Phase 5 (v1.0 complete)
**Requirements**: THEME-01, THEME-02, THEME-03, THEME-04
**Success Criteria** (what must be TRUE):
  1. App kører altid i dark mode uanset system-indstilling
  2. Alle UI-komponenter bruger Gruvbox-farver fra centraliseret tema-system
  3. Ingen hardcodede systemfarver (.blue, .green, .gray) tilbage i kodebasen
  4. Status badges bruger én parametriseret komponent med konsistente Gruvbox-farver
  5. Interactive elements (List selection, Button states) bevarer macOS native feel
**Plans:** 3 plans

Plans:
- [x] 06-01-PLAN.md — Gruvbox Dark color palette, forced dark mode, immersive window
- [x] 06-02-PLAN.md — Badge consolidation and system color migration
- [x] 06-03-PLAN.md — Gap closure: .secondary migration + sidebar selection highlight

#### Phase 7: Animated Progress Rings
**Goal**: Cirkulære progress-ringe med smooth animationer erstatter lineære bars
**Depends on**: Phase 6 (theme colors needed)
**Requirements**: PROG-01, PROG-02, PROG-03
**Success Criteria** (what must be TRUE):
  1. Fase-kort viser animeret cirkulær progress-ring i stedet for lineær bar
  2. Progress-ring animerer smooth ved første visning (ikke ved hver FSEvents-opdatering)
  3. Sidebar projekt-rækker viser mini progress-ring
  4. Ingen performance-degradation med 10+ synlige ringe på skærmen
**Plans:** 2 plans

Plans:

- [x] 07-02-PLAN.md — Sidebar animated bars + card styling + visual verification
