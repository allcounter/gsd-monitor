<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete Phase 1 foundation: Xcode project with Swift 6 strict concurrency, 6 Codable models (Project, Roadmap, Phase, State, Requirement, Plan), and NavigationSplitView skeleton with empty states and automatic dark/light mode support.
  </what-built>
  <how-to-verify>
**1. Launch app and verify basic functionality:**
   - Open `./GSDMonitor.xcodeproj` in Xcode
   - Press Cmd+R to build and run
   - Verify sidebar shows "No Projects Found" with folder icon
   - Verify detail pane shows "Select a Project" with sidebar icon
   - App should launch without crashes or errors

**2. Test theme switching (CRITICAL for Success Criterion 1):**
   - With app running, open macOS System Settings → Appearance
   - Switch between "Light" and "Dark" appearance
   - Expected: App updates instantly (within 1 second) without manual refresh
   - Verify text is readable in both modes (black on white in light, white on dark in dark)
   - Check sidebar, detail pane, and ContentUnavailableView icons all adapt correctly

**3. Memory leak testing with Instruments (CRITICAL for Success Criterion 5):**
   - In Xcode, select Product → Profile (Cmd+I)
   - Choose "Leaks" template
   - Click Record button (red circle)
   - Perform these actions in the running app:
     a. Let app sit idle for 10 seconds (baseline memory)
     b. Switch macOS appearance Light → Dark → Light (5 times)
     c. Let app sit idle for 10 seconds
   - In Instruments, check:
     - "Leaks" panel should show ZERO leaked allocations
     - Memory graph should be flat (no continuous growth)
     - No "Leaked allocation" entries in the list
   - Expected: Clean bill of health, no memory leaks
   - **If leaks detected:** This is a blocker - likely @Observable + @State issue (see PITFALLS.md Pitfall 2)

**4. Swift 6 concurrency verification:**
   - In Xcode, open any Swift file
   - Check Issues navigator (Cmd+5)
   - Expected: ZERO warnings about actor isolation or concurrency
   - Build Settings → Swift Compiler - Language → Swift Language Version should show "Swift 6"
   - Build Settings → Swift Compiler - Language → Strict Concurrency Checking should show "Complete"

**5. Verify all model files exist:**
   - In Xcode Project Navigator, expand GSDMonitor → Models folder
   - Confirm all 6 files present:
     * Project.swift
     * Roadmap.swift
     * Phase.swift
     * State.swift
     * Requirement.swift
     * Plan.swift
   - Open each file, verify `struct` declaration includes `Sendable` conformance
   - Example: `struct Project: Identifiable, Codable, Sendable {`

**6. Verify Xcode previews work:**
   - Open `GSDMonitor/Views/ContentView.swift`
   - Click "Resume" button in preview canvas (or Cmd+Option+P)
   - Verify "Light Mode" preview shows light theme
   - Verify "Dark Mode" preview shows dark theme
   - Both should render without errors

**Success indicators:**
✅ App launches and displays empty states correctly
✅ Theme switching works instantly when macOS appearance changes
✅ Instruments Leaks shows zero leaks after navigation/theme changes
✅ Zero Swift 6 concurrency warnings in build
✅ All 6 models exist with Sendable conformance
✅ Xcode previews render in both light and dark modes

**Failure indicators (report these if found):**
❌ App crashes on launch
❌ Theme switching doesn't work or requires app restart
❌ Instruments shows "Leaked allocation" entries
❌ Compiler warnings about "Main actor-isolated" or "data race"
❌ Missing model files or missing Sendable conformance
❌ Previews fail to render or show errors
  </how-to-verify>
