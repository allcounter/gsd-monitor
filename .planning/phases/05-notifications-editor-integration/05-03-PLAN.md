  <how-to-verify>
**Step 1: Build and launch**
- Build and run the app from Xcode (Cmd+R)
- Verify app launches without crashes

**Step 2: Settings window (EDIT-02, EDIT-03)**
- Press Cmd+, to open Settings
- Verify two tabs: "Editor" and "Notifications"
- Editor tab: Verify your installed editors are listed (Cursor, VS Code, Zed — whichever you have)
- Select your preferred editor from the picker
- Switch to Notifications tab: Verify toggle is ON by default, permission status shown
- Close Settings, reopen — verify preferred editor selection persisted

**Step 3: Editor integration (EDIT-01)**
- Select any project in the sidebar
- Look for "Open in Editor" button in the project detail header
- Click it — verify your preferred editor opens with the project folder
- Click a phase card to open phase detail sheet
- Look for "Open in Editor" button in the phase detail header
- Click it — verify editor opens

**Step 4: Notification permission (NOTF-03)**
- If this is the first time: macOS should prompt for notification permission when a phase change is detected
- Grant permission when prompted
- Check System Settings > Notifications > GSD Monitor — verify the app appears

**Step 5: Live notification test (NOTF-01, NOTF-02)**
- Keep the app running
- In terminal, navigate to any GSD project's .planning/ directory
- Edit ROADMAP.md: change a phase from `- [ ]` to `- [x]` (simulating phase completion)
- Save the file
- Within 1-2 seconds: a macOS notification should appear with the phase status change
- Undo the edit and save again — another notification should appear

**Step 6: Notification toggle**
- Open Settings (Cmd+,), Notifications tab
- Toggle notifications OFF
- Edit ROADMAP.md again — NO notification should appear
- Toggle back ON — notifications resume

**Step 7: Focus mode respect (NOTF-03)**
- If comfortable: enable Focus mode on macOS (System Settings > Focus)
- Trigger a file change — notification should still appear (timeSensitive level)
- Disable Focus mode when done
