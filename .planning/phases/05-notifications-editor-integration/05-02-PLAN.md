Create `Views/Settings/` directory.

Create `SettingsView.swift`:
- `struct SettingsView: View` with `TabView` containing `EditorSettingsView` and `NotificationSettingsView`.
- Tab items: "Editor" with `systemImage: "chevron.left.forwardslash.chevron.right"`, "Notifications" with `systemImage: "bell"`.
- CRITICAL: Add `.frame(width: 450, height: 300)` to TabView — without this the Settings window will be tiny.

Create `EditorSettingsView.swift`:
- Instantiate `EditorService` as `@SwiftUI.State private var editorService = EditorService()`.
- `@AppStorage("preferredEditorID") private var preferredEditorID: String = ""`.
- Show a `Form` with `.formStyle(.grouped)`:
  - `Picker("Preferred Editor:", selection: $preferredEditorID)` with `Text("System Default").tag("")` and a `ForEach` over `editorService.installedEditors` using `Text(editor.name).tag(editor.id)`.
  - Below picker, show "Detected Editors" section listing each installed editor name + path. If no editors detected, show "No supported editors found in /Applications".
  - Add a "Refresh" button that calls `editorService.detectInstalledEditors()`.
- `.onAppear { editorService.detectInstalledEditors() }`.
- Add `.padding()` for proper spacing.

Create `NotificationSettingsView.swift`:
- `@SwiftUI.State private var notificationsEnabled: Bool = UserDefaults.standard.object(forKey: "notificationsEnabled") as? Bool ?? true`.
- `@SwiftUI.State private var permissionStatus: String = "Checking..."`.
- Show a `Form` with `.formStyle(.grouped)`:
  - `Toggle("Enable Notifications", isOn: $notificationsEnabled)` with `.onChange` that writes to UserDefaults: `UserDefaults.standard.set(newValue, forKey: "notificationsEnabled")`.
  - Text explaining: "Receive notifications when phase statuses change in your GSD projects."
  - Section "Permission Status" showing the current macOS notification permission state (authorized/denied/not determined).
  - If denied, show text: "Notifications are blocked by macOS. Enable in System Settings > Notifications > GSD Monitor."
- `.task { await checkPermissionStatus() }` — reads `UNUserNotificationCenter.current().notificationSettings()` and updates `permissionStatus` string.
- Add `.padding()`.

Modify `GSDMonitorApp.swift`:
- Add `Settings { SettingsView() }` scene after the WindowGroup scene.
- This automatically enables Cmd+, shortcut in the menu bar.

Add all three new view files to Xcode project using ruby xcodeproj gem.

<task type="auto">
  <name>Task 2: Wire NotificationService lifecycle and clickable file paths</name>
  <files>GSDMonitor/Views/ContentView.swift, GSDMonitor/Views/Dashboard/PhaseDetailView.swift, GSDMonitor/Views/DetailView.swift</files>
  <action>
Modify `ContentView.swift`:
- Add `@SwiftUI.State private var notificationService: NotificationService?` (optional because it needs projectService to init).
- In the existing `.task` block, after `await projectService.loadProjects()`, create and start the notification service:
  ```swift
  let ns = NotificationService(projectService: projectService)
  notificationService = ns
  ns.startMonitoring()
  ```
- In the existing `.onChange(of: scenePhase)` handler, no changes needed (NotificationService monitors via withObservationTracking which follows ProjectService changes).
- Add `.onDisappear { notificationService?.stopMonitoring() }` to clean up.

Modify `PhaseDetailView.swift`:
- Add `@SwiftUI.State private var editorService = EditorService()` at the top of `PhaseDetailView`.
- In the private `PlanCard` struct, the tasks currently show task name only. Plan.Task has a `name` property. The file paths that users want to click are in the plan's context — but the Plan model only stores `objective` and `tasks` (no file paths directly).
- Instead: Add an "Open in Editor" button at the plan level (not task level). Since each plan's `objective` describes what files it touches, and the PLAN.md files themselves are in `.planning/phases/`, add a button to each PlanCard that opens the corresponding PLAN.md file:
  - The plan has `phaseNumber` and `planNumber`. The project has `path`. Construct the file path: `project.path/.planning/phases/{phaseDir}/{phaseNum}-{planNum}-PLAN.md`.
  - BUT: the phase directory name is not stored in the Plan model. A simpler approach: Add a small "Open in Editor" (folder icon) button to the PhaseDetailView header that opens the project's `.planning/` directory in the preferred editor (editors like VS Code/Cursor open folders).
  - Add to the PhaseDetailView header (next to the "Done" button): A button with `systemImage: "arrow.up.forward.app"` labeled "Open in Editor" that calls `editorService.openFile(project.path)`. This opens the entire project folder in the editor — the most useful action since developers want to see the full project context.
- Also add an `.onAppear { editorService.detectInstalledEditors() }` to PhaseDetailView.
- In the main `DetailView.swift`, add a similar "Open in Editor" button in the project header area (next to project name). Add `@SwiftUI.State private var editorService = EditorService()` and a button calling `editorService.openFile(project.path)`. Only show this button if `editorService.installedEditors` is not empty.

NOTE: Use `_Concurrency.Task` (not bare `Task`) anywhere Task is used to avoid collision with the Plan.Task model that is imported.
