  <name>Task 1: Create Editor model and EditorService</name>
  <files>GSDMonitor/Models/Editor.swift, GSDMonitor/Services/EditorService.swift</files>
  <action>
Create `Editor.swift` in Models/:
- `struct Editor: Identifiable, Codable, Sendable` with properties: `id: String` (bundle identifier), `name: String`, `path: URL`.
- Custom Codable init/encode for URL (same pattern as Project.swift - encode path as String, decode with URL(fileURLWithPath:)).

Create `EditorService.swift` in Services/:
- `@MainActor @Observable final class EditorService`.
- Property: `var installedEditors: [Editor] = []`.
- Property: `@AppStorage("preferredEditorID") var preferredEditorID: String = ""` - NOTE: @AppStorage does NOT work in @Observable classes. Instead, use a computed property backed by UserDefaults: `var preferredEditorID: String { get { UserDefaults.standard.string(forKey: "preferredEditorID") ?? "" } set { UserDefaults.standard.set(newValue, forKey: "preferredEditorID") } }`.
- Computed: `var preferredEditor: Editor? { installedEditors.first { $0.id == preferredEditorID } }`.
- Hardcoded known editors dictionary: `["com.microsoft.VSCode": "Visual Studio Code", "com.todesktop.230313mzl4w4u92": "Cursor", "dev.zed.Zed": "Zed"]`.
- `func detectInstalledEditors()`: Enumerate both `/Applications` and `~/Applications` (using FileManager). For each .app bundle found, check `Bundle(url:)?.bundleIdentifier` against known editors. Populate `installedEditors`. Use `.skipsPackageDescendants` and `.skipsHiddenFiles` options on enumerator.
- `func openFile(_ fileURL: URL)`: Opens file in preferred editor (or first installed if no preference). Uses `NSWorkspace.shared.open([fileURL], withApplicationAt: editor.path, configuration: NSWorkspace.OpenConfiguration())` with async/await completion handler. Print error to console if launch fails. Guard against no editors installed.
- `func openFile(_ fileURL: URL, with editor: Editor)`: Opens file in a specific editor using same NSWorkspace pattern.
- Call `detectInstalledEditors()` in an `init()` or provide a `refreshEditors()` method.

Add both files to the Xcode project using: `ruby -r xcodeproj -e '...'` (same pattern used in phases 01-04 to add Swift files to GSDMonitor target).

<task type="auto">
  <name>Task 2: Create NotificationService and AppDelegate</name>
  <files>GSDMonitor/Services/NotificationService.swift, GSDMonitor/App/AppDelegate.swift, GSDMonitor/App/GSDMonitorApp.swift</files>
  <action>
Create `AppDelegate.swift` in App/:
- `@MainActor final class AppDelegate: NSObject, NSApplicationDelegate, @preconcurrency UNUserNotificationCenterDelegate`.
- `import UserNotifications` at top.
- In `applicationDidFinishLaunching(_:)`: set `UNUserNotificationCenter.current().delegate = self`.
- Implement `nonisolated func userNotificationCenter(_:willPresent:) async -> UNNotificationPresentationOptions` returning `[.banner, .sound]` (show notifications even when app is in foreground).

Create `NotificationService.swift` in Services/:
- `import UserNotifications` at top.
- `@MainActor @Observable final class NotificationService`.
- Property: `private let projectService: ProjectService` (injected via init).
- Property: `private var previousPhaseStates: [String: PhaseStatus] = [:]` (keyed by "projectName-phaseNumber" for uniqueness).
- Property: `private var monitoringTask: _Concurrency.Task<Void, Never>?` (use _Concurrency.Task to avoid Plan.Task collision).
- Property: `var notificationsEnabled: Bool { get { UserDefaults.standard.bool(forKey: "notificationsEnabled") } set { UserDefaults.standard.set(newValue, forKey: "notificationsEnabled") } }` - default to true via registering defaults.
- `init(projectService: ProjectService)`: Store reference and register UserDefaults defaults `["notificationsEnabled": true]`.
- `func requestPermissionIfNeeded() async -> Bool`: Check `UNUserNotificationCenter.current().notificationSettings()`. If `.notDetermined`, call `requestAuthorization(options: [.alert, .sound])`. Return true if authorized.
- `func startMonitoring()`: Cancel existing monitoringTask. Create new `_Concurrency.Task` that loops using `withObservationTracking` to watch `projectService.projects`. On change callback, call `checkForChanges()`. Snapshot initial states on first run.
- `func stopMonitoring()`: Cancel monitoringTask, set to nil.
- `private func snapshotCurrentStates()`: Iterate all projects/phases, populate `previousPhaseStates` with current statuses.
- `private func checkForChanges()`: Compare current project/phase statuses against `previousPhaseStates`. For each status that changed (and previousPhaseStates had an entry for it - skip brand new phases), call `scheduleNotification(...)`. Update previousPhaseStates after checking. Only schedule if `notificationsEnabled` is true.
- `private func scheduleNotification(projectName: String, phaseName: String, phaseNumber: Int, newStatus: PhaseStatus)`: Call `requestPermissionIfNeeded()` first. Build `UNMutableNotificationContent` with title "Phase \(phaseNumber) - \(statusText)" and body "\(projectName): \(phaseName)". Set `.interruptionLevel = .timeSensitive` for Focus mode pass-through. Use `trigger: nil` for immediate delivery. Add via `UNUserNotificationCenter.current().add(request)`.

Modify `GSDMonitorApp.swift`:
- Add `import UserNotifications` (needed if not already imported).
- Add `@NSApplicationDelegateAdaptor(AppDelegate.self) var appDelegate` property.
- Keep existing WindowGroup scene (do NOT add Settings scene yet - that is Plan 02).

Add new files to Xcode project using ruby xcodeproj gem.
