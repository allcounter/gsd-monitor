  <action>
    **BookmarkService.swift:**

    Create a `final class BookmarkService: Sendable` that manages security-scoped bookmark lifecycle. Use an `nonisolated` approach with UserDefaults for storage. Key methods:

    - `func saveBookmark(for url: URL, identifier: String) throws` — Creates bookmark data with `.withSecurityScope` option and stores in UserDefaults with key "bookmark_{identifier}".
    - `func resolveBookmark(for identifier: String) throws -> URL?` — Resolves stored bookmark data. Checks `bookmarkDataIsStale` and refreshes automatically if stale. Returns nil if no bookmark stored.
    - `func removeBookmark(for identifier: String)` — Removes bookmark data from UserDefaults.
    - `func allBookmarkIdentifiers() -> [String]` — Returns all stored bookmark identifiers (filter UserDefaults keys by "bookmark_" prefix).
    - `func accessSecurityScoped<T>(_ url: URL, operation: () throws -> T) throws -> T` — Wraps start/stopAccessingSecurityScopedResource around an operation. Throws if access denied.

    Use a private let for the UserDefaults suite name (e.g., "com.gsdmonitor.bookmarks") to isolate bookmark storage. Define a `BookmarkError` enum with cases: `accessDenied`, `staleAndUnrefreshable`, `creationFailed(Error)`.

    Since UserDefaults is thread-safe for reads/writes, and all stored types are Data (value types), this class can be Sendable. Mark with `@unchecked Sendable` if needed due to UserDefaults reference, but prefer structuring so the compiler accepts it.

    **ProjectDiscoveryService.swift:**

    Create a `struct ProjectDiscoveryService: Sendable` with:

    - `func discoverProjects(in rootURL: URL) async -> [DiscoveredProject]` — Uses FileManager.DirectoryEnumerator with options `[.skipsHiddenFiles, .skipsPackageDescendants]`. For each URL in enumeration:
      - Check `.isSymbolicLinkKey` resource value; skip symlinks entirely (call `skipDescendants()`)
      - Skip directories named: "node_modules", ".git", "build", "DerivedData", ".build", "Pods", "Carthage"
      - When `lastPathComponent == ".planning"` AND the directory contains ROADMAP.md → found a project. Append parent URL. Call `skipDescendants()` (don't recurse into .planning/).
      - Use `enumerator.level` to enforce depth limit of 6 (skip descendants beyond depth 6).
      - Error handler: log error, return true (continue enumeration).

    - `func discoverProjects(in rootURLs: [URL]) async -> [DiscoveredProject]` — Calls single-root version for each URL, combines results, removes duplicates by path.

    Define a `DiscoveredProject` struct (Sendable):
    ```swift
    struct DiscoveredProject: Sendable, Equatable {
        let name: String       // directory name (last path component of parent)
        let path: URL          // project root (parent of .planning/)
        let scanSource: URL    // which root directory found this project
    }
    ```

    The scanning must NOT block the main thread. Use `Task.detached` internally for the FileManager enumeration work so it runs on a background thread. Return results via async.

    Add both files to the Xcode project target in project.pbxproj.
  </action>
