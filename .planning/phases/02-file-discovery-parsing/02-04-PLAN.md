  <action>
    Create `@MainActor @Observable final class ProjectService` that orchestrates all services:

    **Properties:**
    - `var projects: [Project] = []` — All discovered + manually added projects
    - `var scanSources: [URL]` — Root directories to scan (default: [~/Developer]). Stored in UserDefaults.
    - `private let discoveryService = ProjectDiscoveryService()`
    - `private let bookmarkService = BookmarkService()`
    - `private let roadmapParser = RoadmapParser()`
    - `private let stateParser = StateParser()`
    - `private let requirementsParser = RequirementsParser()`
    - `private let planParser = PlanParser()`
    - `private let configParser = ConfigParser()`

    **Lifecycle:**
    - `func loadProjects() async` — Called on app launch. Steps:
      1. Load manually added projects from bookmarks (BookmarkService.allBookmarkIdentifiers)
      2. Resolve each bookmark, parse project files, add to projects array
      3. Scan each scanSource directory with ProjectDiscoveryService
      4. For each discovered project, parse all .planning/ files:
         - Read ROADMAP.md -> RoadmapParser.parse() -> Roadmap
         - Read STATE.md -> StateParser.parse() -> State
         - Read REQUIREMENTS.md -> RequirementsParser.parse() -> [Requirement] (store in Project if model supports, or in a separate property)
         - Read config.json -> ConfigParser.parse() -> PlanningConfig
         - Read all PLAN.md files in phases/ subdirectories -> PlanParser.parse() -> [Plan]
      5. Create Project struct for each, add to projects array
      6. Remove duplicates (same path from both scan and manual add)
      7. Handle disappeared projects: if a bookmarked project's directory no longer exists, remove its bookmark

    **Manual project management:**
    - `func addProjectManually() async` — Shows NSOpenPanel for directory selection:
      - `canChooseDirectories = true`, `canChooseFiles = false`, `allowsMultipleSelection = false`
      - Validate selected directory has .planning/ subfolder. If not, show no alert (just don't add).
      - Validate .planning/ contains at minimum ROADMAP.md. If not, don't add.
      - Save bookmark via BookmarkService
      - Parse project files, add to projects array
    - `func removeManualProject(_ project: Project)` — Removes from projects array and deletes bookmark

    **Helper methods:**
    - `private func parseProject(at url: URL, scanSource: URL?) -> Project?` — Reads and parses all .planning/ files at the given project root. Returns nil if parsing fails entirely. Partial parse results are OK (e.g., missing STATE.md -> state = nil).
    - `private func readFileContent(at url: URL) -> String?` — Reads file as UTF-8 string, returns nil on error.

    **Project grouping:**
    - `var groupedProjects: [(source: String, projects: [Project])]` — Computed property that groups projects by their scan source. Manual projects use "Manually Added" as source. Scanned projects use display-friendly path (e.g., "~/Developer").
    - Sort groups: scan sources first (alphabetical), "Manually Added" last.
    - Sort projects within groups: alphabetical by name.

    Store which project IDs are manually added vs scanned in UserDefaults (Set<String> of project path strings).

    NOTE: Use `@Observable` (macOS 14+) per Observation framework, NOT `ObservableObject`/@Published. This avoids the @Published memory leak pitfall and is the modern SwiftUI pattern. Use `@SwiftUI.State` in views to avoid name collision with State model.
  </action>

    Uses @Observable, not ObservableObject.
  </verify>
