   enum StatusFilter: String, CaseIterable {

   - Add `.searchScopes($statusFilter)` with ForEach over StatusFilter.allCases rendering Text(filter.rawValue).tag(filter)

5. Update `emptyState` check: show empty state only if `projectService.projects.isEmpty` (not filtered), and show "No matching projects" ContentUnavailableView when `filteredProjects` is empty but `projectService.projects` is not.

6. Performance note: List already handles 100+ items efficiently per research. Do NOT use LazyVStack. Do NOT add .id() modifier. Keep ProjectRow simple.

Create `GSDMonitor/Views/CommandPalette/` directory.

**CommandPaletteView.swift:**

1. Define `CommandResult` enum (Identifiable, conforming to Hashable for List):
   ```swift
   enum CommandResult: Identifiable, Hashable {
       case project(Project)
       case phase(Phase, projectName: String, projectID: UUID)
       case requirement(Requirement, projectName: String, projectID: UUID)

       var id: String { ... }  // unique string per case
   }
   ```
   Note: Phase and Requirement are Identifiable+Codable structs (already Sendable). For Hashable, implement hash(into:) using the id.

2. Create CommandPaletteView:
   - Properties: `let projectService: ProjectService`, `let onSelectProject: (UUID) -> Void`
   - @Environment(\.dismiss) private var dismiss
   - @SwiftUI.State private var query = ""
   - @FocusState private var searchFocused: Bool

3. Computed property `searchResults: [CommandResult]`:
   - If query is empty, return empty array (or first 10 projects as suggestions)
   - Search projects by name using localizedCaseInsensitiveContains
   - Search phases by name across all projects
   - Search requirements by id or description across all projects
   - Return combined results, projects first, then phases, then requirements
   - Limit to 20 results total to keep UI snappy

4. Layout:
   - VStack(spacing: 0):
     - Search field: HStack with magnifyingglass Image + TextField("Search projects, phases, requirements...", text: $query) with .textFieldStyle(.plain) + .focused($searchFocused) + clear button (xmark.circle.fill) when query non-empty
     - .padding() on the HStack, .background(Color(nsColor: .controlBackgroundColor))
     - Divider
     - Results: if searchResults.isEmpty, show ContentUnavailableView("No Results"). Otherwise List of CommandResultRow views.
     - Each row: onTapGesture that calls onSelectProject(projectID) then dismiss()

5. CommandResultRow (private struct):
   - HStack: icon (folder/list.bullet.rectangle/checkmark.circle) + VStack(title, subtitle) + Spacer + breadcrumb
   - Extract projectID from each CommandResult case for navigation

6. Modifiers:
   - .frame(width: 600, height: 400)
   - .onAppear { searchFocused = true }

**ContentView.swift updates:**

1. Add state: `@SwiftUI.State private var showCommandPalette = false`

2. After the NavigationSplitView, add:
   ```swift
   .sheet(isPresented: $showCommandPalette) {
       CommandPaletteView(
           projectService: projectService,
           onSelectProject: { projectID in
               selectedProjectID = projectID
           }
       )
   }
   .background {
       Button("") {
           showCommandPalette = true
       }
       .keyboardShortcut("k", modifiers: .command)
       .hidden()
   }
   ```

3. Place the .background modifier on the NavigationSplitView (inside the view hierarchy, not outside). The hidden button must be in the active view hierarchy for the keyboard shortcut to register.
