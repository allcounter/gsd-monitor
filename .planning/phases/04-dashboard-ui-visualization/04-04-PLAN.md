**What was built:**
Complete Phase 4 dashboard UI: phase cards with markdown rendering, progress bars, requirement badges, requirement detail sheets, phase drill-down with plans/tasks, searchable sidebar with status filter, and Cmd+K command palette.

**Verification steps for the user:**

1. **Phase Cards (ROAD-01, ROAD-02, ROAD-04):**
   - Select a project with a roadmap (e.g., gsd-monitor itself)
   - Verify phase cards show: phase name, goal text, status badge, progress bar with percentage
   - Verify markdown renders in goal text (bold shows as bold, not raw **text**)
   - Verify overall project progress bar appears above phase cards

2. **Phase Drill-Down (ROAD-03):**
   - Click on a phase card
   - Verify a sheet opens showing: phase goal, requirements, and plan list
   - Verify each plan shows: plan number, objective, task list with status icons
   - Close the sheet

3. **Requirement Badges (REQ-01, REQ-02, REQ-03):**
   - On a phase card, look for colored REQ-ID badges (e.g., "NAV-01", "UI-01")
   - Click a requirement badge
   - Verify a sheet opens showing: requirement ID, category, definition, mapped phases, related plans, and status
   - Verify the "Related Plans" section shows cross-reference between requirement and plans
   - Close the sheet

4. **Search/Filter (NAV-03):**
   - Type a project name in the sidebar search field
   - Verify the project list filters as you type
   - Try the status filter scopes (All, Active, Completed)
   - Clear search and verify all projects return

5. **Command Palette (NAV-04):**
   - Press Cmd+K
   - Verify a modal search palette appears with auto-focused search field
   - Type a project name — verify matching results appear
   - Type a phase name — verify phase results appear
   - Type a requirement ID (e.g., "NAV") — verify requirement results appear
   - Click a result — verify the palette closes and the project is selected in sidebar

6. **Performance:**
   - If you have many projects in ~/Developer, verify sidebar scrolls smoothly
   - No visible lag when switching between projects

7. **Dark/Light Mode:**
   - Toggle macOS appearance — verify cards, badges, and palette adapt correctly
