import Foundation
import CoreServices

The service must be `@MainActor final class` (same pattern as ProjectService) to safely update state and interact with SwiftUI.

**Core API:**
- `func watch(paths: [URL]) -> AsyncStream<[URL]>` -- Creates FSEventStream watching the given .planning/ paths. Returns an AsyncStream that yields arrays of changed URLs. The stream uses a C callback that:
  1. Reads `numEvents` paths from the `eventPaths` parameter (cast to `NSArray as! [String]`)
  2. Filters out any path containing `"/.git/"` -- this prevents event storms during Git operations
  3. Converts remaining paths to `URL(fileURLWithPath:)`
  4. Yields non-empty arrays via the AsyncStream continuation

- `func stopWatching()` -- Executes the three-step cleanup sequence:
  1. `FSEventStreamStop(stream)` -- stop receiving events
  2. `FSEventStreamInvalidate(stream)` -- unschedule from run loop
  3. `FSEventStreamRelease(stream)` -- release memory
  Then finishes the AsyncStream continuation and nils out stored references.

- `func updatePaths(_ paths: [URL])` -- Stops current stream and creates a new one with updated paths. Used when projects are added/removed.

**FSEventStream configuration:**
- `latency`: 1.0 second (research recommendation: start with Apple's example value)
- `flags`: `kFSEventStreamCreateFlagUseCFTypes | kFSEventStreamCreateFlagNoDefer` -- UseCFTypes gives NSArray in callback (easier to work with), NoDefer ensures first event fires immediately
- Do NOT use `kFSEventStreamCreateFlagFileEvents` initially (research recommendation: directory-level events are sufficient, fewer events)
- `sinceWhen`: `FSEventsGetCurrentEventId()` -- only future events
- Schedule on `CFRunLoopGetMain()` with `kCFRunLoopDefaultMode` (since service is @MainActor, main run loop is appropriate)

**Passing continuation to C callback:**
The FSEventStreamCallback is a C function pointer and cannot capture Swift values. Use the `context` parameter:
1. Store the `AsyncStream<[URL]>.Continuation` in an `UnsafeMutablePointer` allocated with `.allocate(capacity: 1)` and initialized
2. Pass it as `clientCallBackInfo` in `FSEventStreamContext`
3. In the callback, retrieve via `Unmanaged<AnyObject>.fromOpaque(clientCallBackInfo!).takeUnretainedValue()` or cast from the raw pointer
4. On cleanup, deallocate the pointer

**IMPORTANT Swift 6 considerations:**
- The FSEventStreamCallback is `@convention(c)` -- it cannot capture `self` or any non-trivial Swift values
- All communication from callback to Swift must go through the continuation pointer
- The continuation pointer must remain valid for the lifetime of the stream
- Use `nonisolated(unsafe)` if needed for the pointer storage, since it is only accessed from MainActor methods and the C callback (which is serialized by FSEvents)

**Property storage:**
- `private var stream: FSEventStreamRef?`
- `private var continuation: AsyncStream<[URL]>.Continuation?`
- `private var continuationPointer: UnsafeMutablePointer<AsyncStream<[URL]>.Continuation>?` -- allocated/deallocated with stream lifecycle
