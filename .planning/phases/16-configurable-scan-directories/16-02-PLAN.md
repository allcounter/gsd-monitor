```swift
    HStack {
        Spacer()

            showingScanSettings.toggle()
        } label: {
            Image(systemName: "gearshape")
                .foregroundStyle(Theme.textSecondary)
        }
        .buttonStyle(.plain)
        .popover(isPresented: $showingScanSettings, arrowEdge: .bottom) {
            ScanDirectoriesPopoverView(projectService: projectService)
                .frame(width: 340)
        }
        .padding(8)
    }
    .background(Theme.bg0)

```

3. **Replace Section with DisclosureGroup** in `projectSection(for:)`:
- Use `DisclosureGroup` with a binding derived from `expandedGroups` set
- All groups start expanded (default). A group is collapsed when its key IS in `expandedGroups`
- Create a binding:
```swift
let isExpanded = Binding<Bool>(
    get: { !expandedGroups.contains(group.source) },
    set: { newValue in
        withAnimation {
            if newValue {
                expandedGroups.remove(group.source)
            } else {
                expandedGroups.insert(group.source)
            }
        }
    }
)
```
- Replace `Section(header: Text(group.source))` with:
```swift
DisclosureGroup(isExpanded: isExpanded) {
    ForEach(group.projects) { project in
        ProjectRow(...)
            .tag(project.id)
            .listRowBackground(Theme.bg0)
            .listRowSeparator(.hidden)
    }
} label: {
    Text(group.source)
        .font(.caption)
        .foregroundStyle(Theme.textMuted)
}
```

4. **Handle empty groups:**
- In `filteredProjects`, ensure groups with 0 projects are still included when no search/filter is active
- Modify groupedProjects usage: when `searchText.isEmpty && statusFilter == .all`, include all scan sources even if they have 0 projects
- For empty groups, show a "No projects found" text inside the DisclosureGroup:
```swift
if group.projects.isEmpty {
    Text("No projects found")
        .font(.caption)
        .foregroundStyle(Theme.fg4)
        .italic()
        .padding(.vertical, 4)
        .padding(.leading, 8)
}
```

5. **Ensure ~/Developer is always first:**
- The existing `groupedProjects` sort already handles this (scan sources alphabetically, "Manually Added" last)
- Verify ~/Developer sorts first since `~` sorts after letters — if needed, special-case ~/Developer to always be first in the sort
