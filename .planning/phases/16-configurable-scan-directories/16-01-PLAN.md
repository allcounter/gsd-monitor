<task type="auto">
  <name>Task 1: Add scan state tracking and add/remove methods to ProjectService</name>
  <files>GSDMonitor/Services/ProjectService.swift</files>
  <action>
Add per-source scan state tracking and targeted add/remove methods to ProjectService:

1. **Add ScanSourceState struct** (inside ProjectService or at file scope):
```swift
struct ScanSourceState {
    var isScanning: Bool = false
    var lastScannedAt: Date? = nil
    var isAccessible: Bool = true
}
```

2. **Add observable properties:**
- `var scanSourceStates: [String: ScanSourceState] = [:]` — keyed by URL.path
- `var duplicateWarningPath: String? = nil` — transient state for "Already added" warning in popover

3. **Add `addScanDirectory(_ url: URL)` method:**
- Guard duplicate: if `scanSources.contains(where: { $0.path == url.path })`, set `duplicateWarningPath = url.path` and return
- Append to `scanSources` (triggers UserDefaults write)
- Check accessibility: `FileManager.default.fileExists(atPath: url.path, isDirectory: &isDir)` — set `isAccessible` accordingly
- Set `scanSourceStates[url.path].isScanning = true`
- In a `_Concurrency.Task`: discover projects in the new URL via `discoveryService.discoverProjects(in: [url])`, parse each, append to `projects` (skip duplicates by path), then set `isScanning = false`, `lastScannedAt = Date()`
- After scan: call `stopMonitoring()` then `startMonitoring()` then nil `scanSourceMonitoringTask` and call `startScanSourceMonitoring()` — this restarts FSEvents to include the new directory (per pitfall #1 in research)

4. **Add `removeScanDirectory(_ url: URL)` method:**
- Guard: do NOT allow removing the default ~/Developer source
- Remove projects whose `path.path.hasPrefix(url.path)` AND are NOT in `manualProjectPaths`
- Remove from `scanSources`
- Remove from `scanSourceStates`
- Restart monitoring (same stop/start pattern as add)

5. **Add `addScanDirectoryViaPanel()` async method** (for the + button in the popover):
- Create `NSOpenPanel` with `canChooseDirectories = true`, `canChooseFiles = false`, `allowsMultipleSelection = false`, `message = "Select a directory to scan for GSD projects"`
- `await panel.begin()`, guard `.OK` and `panel.url`
- Call `addScanDirectory(url)`

6. **Update `loadProjects()` to populate initial `scanSourceStates`:**
- After scanning completes for each source, set `lastScannedAt = Date()`, `isScanning = false`, `isAccessible = true`
- Check each scan source for accessibility and set `isAccessible` accordingly

7. **Update `groupedProjects`:**
- Ensure ~/Developer group is always first (already done via sort)
- Ensure projects within each group are sorted alphabetically (already done)
- No other changes needed — the existing grouping logic works correctly

IMPORTANT: Use `_Concurrency.Task` (not bare `Task`) everywhere in this file — GSDMonitor.Task shadows Swift's Task in this module.

  <action>
Create a new SwiftUI view `ScanDirectoriesPopoverView` in the Settings directory. This is the popover UI for managing scan directories.

**Structure:**
```
VStack(alignment: .leading, spacing: 0)
  - Header: "Scan Directories" (.headline font, padded)
  - Divider
  - ScrollView (for directory list):
    - "Default" section label (.caption, Theme.textMuted, padded)
    - ScanDirectoryRow for ~/Developer (not removable)
    - Divider
    - "Additional" section label (.caption, Theme.textMuted, padded)
    - ForEach additional sources -> ScanDirectoryRow (removable)
    - If no additional sources: "No additional directories" (Theme.fg4, italic, padded)
    - If duplicateWarningPath set: inline warning "Already added" in Theme.brightYellow
  - Divider
  - Bottom bar: + button "Add Directory..." with Label("Add Directory...", systemImage: "plus")
```

**ScanDirectoryRow** (private subview or inline):
- Shows abbreviated path with ~ (e.g., "~/Projects")
- Shows project count: count projects from `projectService.projects` where `project.path.path.hasPrefix(sourceURL.path)` and not in manualProjectPaths
- Shows last scan time using `RelativeDateTimeFormatter` with `.abbreviated` unitsStyle (e.g., "scanned 2 min. ago")
- Shows scanning indicator: `ProgressView()` (small spinner) when `scanSourceStates[path].isScanning == true`
- Shows error badge: small `Image(systemName: "exclamationmark.triangle.fill")` in `Theme.brightYellow` when `scanSourceStates[path].isAccessible == false`
- If removable: small minus/trash button on trailing side

**Path abbreviation:**
```swift
let homePath = FileManager.default.homeDirectoryForCurrentUser.path
let displayPath = url.path.replacingOccurrences(of: homePath, with: "~")
```

**Drag-and-drop support:**
- Apply `.onDrop(of: [.fileURL], isTargeted: $isDropTargeted)` to the entire popover VStack
- In the drop handler: load URL from NSItemProvider, verify `url.hasDirectoryPath`, call `projectService.addScanDirectory(url)`
- When `isDropTargeted` is true, show a subtle highlight border (Theme.brightBlue.opacity(0.3))

**+ button action:**
```swift
_Concurrency.Task {
    await projectService.addScanDirectoryViaPanel()
}
```

**Remove button action:**
```swift
projectService.removeScanDirectory(url)
```

**Relative time refresh:**
- Use `TimelineView(.periodic(from: .now, by: 60))` wrapping the rows that display lastScannedAt — this auto-refreshes every 60 seconds without manual timer management

**Styling:**
- Frame width ~340, popover background Theme.bg0 or Theme.bg1
- Use Theme colors consistently (fg1 for primary text, fg4/textMuted for secondary, textSecondary for labels)
- Compact row height to fit multiple directories comfortably

Register the new file in project.pbxproj if needed (check if the project auto-discovers files or requires manual registration).
